Searching for locators

Use the Find panel (Edit menu) to find records that contain references to specified pages or range of pages.

To ensure that Cindex confines the search to the locator field, and evaluates locators correctly, do the following:

  • Enter the locator or locator range in the box for search text. The page reference may be a multi-part locator. (To find records that contain no locators or badly formed ones, leave the search text empty.)
  • Choose the name of the locator field (usually Page) from the popup menu under Not.
  • Check Evaluate References. This ensures that Cindex ignores cross-references, and will correctly evaluate numbers and multi-part locators. (If Evaluate References is unchecked, Cindex treats the contents of the locator field as ordinary text.)

When searching for locators Cindex understands page ranges. If Cindex sees two locators separated by a hyphen (for example: 57-66) it understands that it should search for locators that fall within a range. It will also find a record in which the locator field contains a sequence such as 50–58, or 63-67. If you are using multi-part locators, you may abbreviate the second reference of the range so that it shows only parts that differ from the first reference.

You may use a character other than hyphen to denote an inclusive range, but only if you have told Cindex what this character is. See Specifying the structure of page references to change the standard connecting character.
