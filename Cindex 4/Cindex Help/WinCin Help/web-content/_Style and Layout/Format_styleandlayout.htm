Style and layout of entries

To set the style and layout of index entries, choose Style & Layout... from the Document menu.

Click Indented or Run-in to specify indented or run-in (paragraph) as the basic index style. If you click Run-in, you can choose from the adjacent drop-down list the level of heading below which Cindex will run-in headings (the default setting is to run-in all levels below the main heading). The choice you make takes effect only when the index is displayed in a fully formatted view. Cindex always uses a standard indented style for display in a draft view.

For each major style, Cindex provides variants you can select through the Variant drop-down list.

Standard Layout

Entries are formatted in the standard way, either indented or run-in.

If you choose an indented style, you can force individual entries to be formatted in run-in style. To force run-on subheadings below a particular level, type a vertical bar (|) as the last character in the heading from which you want lower levels to run-on. Cindex will automatically substitute the correct punctuation for the | when it formats the entry.

Indent Major Subheadings

If the index contains sub-subheadings as well as subheadings Cindex can format entries so that subheadings are run-in from headings, except when any subheading has a sub-subheading beneath it, in which case all the subheadings under the relevant heading are indented and sub-subheadings are run-in.

Run-back Subheadings

Cindex will modify the layout of any entry in which a subheading (or sub-subheading) falls under a heading (or subheading) that contains no immediately following page reference. Cindex converts the subheading (or sub-subheading) to a phrase that modifies the main heading (or subheading).

Show Repeated Headings

Cindex displays every level of heading for each entry, (i.e., it does not suppress repeated headings).

If your entries contain multiple levels of heading, you can collapse heading levels in the formatted index, so that, for example, all subheadings are hidden, and the page references and cross-references that belong to them are merged with those from the main heading. You can collapse headings from any level by checking Collapse Headings below and choosing from the menu the level of heading below which headings will be collapsed.

When you collapse entries, Cindex automatically suppresses "See" (but not "See also") references that originate from a heading level that is collapsed.

If you set Style and Layout when no index is open for work, the settings become the default ones that Cindex uses for new indexes when you create them.
