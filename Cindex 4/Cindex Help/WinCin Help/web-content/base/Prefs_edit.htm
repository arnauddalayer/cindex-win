Settings for adding and editing records

Cindex offers a variety of ways to customize its behavior when you are adding and editing records. To make any of the settings discussed below, choose Preferences... from the Edit menu, then click the Editing tab.

General

  • When you add or edit a record Cindex normally opens a panel at the foot of the main view. If you prefer to work in a separate window when adding and editing records, check Use Separate Window.
  • A draft view is often the best to work with when you add and edit records, because each record is always visible individually. To make Cindex switch to a draft view when you add or edit a record, check Switch to Draft View. If the original view was formatted, Cindex will revert to it when you finish editing. If you want Cindex to retain the fully formatted view while you add or edit records, uncheck Switch to Draft View.
  • You can set Cindex so that it automatically removes superfluous spaces whenever more than one separates words. If you check Remove Extra Spaces, Cindex will remove extra spaces no matter how they the are inserted into records.
  • When exchanging ('flipping') the contents of two record fields, Cindex can make a literal exchange, or can do it intelligently, adjusting leader case and moving articles and prepositions as appropriate to the content of the fields. To enable this intelligent exchange, check Smart Flip.
  • Cindex can generate the second part of a page range automatically when you type the character that connects the two parts of a range. For example, if a dash is the connecting character, and you typed 55- Cindex would complete the reference as 55-56 (you can, of course, then edit the number as required). Cindex does this only if the last recognized component of the last page reference in the locator field is a number in Arabic numerals. To enable this, check Complete Page Ranges.

Adding Records

  • When you add records, Cindex normally displays the most recently-added records in record-number order. To make it show a new record immediately in its sorted position, check Track New Entries.
  • To make Cindex automatically re-use the locator (page reference) from the record you last-entered, check Carry Locators Forward.
  • To make Cindex use an existing heading in the index to complete (if it can) a heading you are typing, check Auto-Complete Entries. By default, Cindex completes headings only if what you type matches exactly (in letter case and style) the text of an existing heading. To permit Cindex to complete headings that differ in letter case or style, check Ignore Style & Case. If you want Cindex to display the existing entry from which it is taking a heading to use for completing a new one, Track Source.

Editing Records

  • You can set Cindex so that when you change a heading, it will automatically make the same change to all immediately succeeding records that shared the original heading. To enable this automatic propagation of changes to succeeding records, check Propagate Changes. Your setting determines how Cindex behaves when you open a record for editing. Your setting establishes the default state of the Propagate button in record window’s toolbar (Cindex sets the button to the default state whenever a new record is placed in the window). You can engage or disengage propagation on a record-by-record basis.
  • As you edit successive records in a series, Cindex adjusts the display in the main view to show your progress through the records. When you finish editing, Cindex normally leaves the display showing the last record you worked on. To make Cindex restore the display to show the first record you worked on, check Return to Entry Point.
  • When you add or change a label, Cindex by default does not mark the record as 'modified' (i.e., it does not update the date stamp and user identifier). To specify that a label change is considered a modification, check Label Sets Change Flag.
  • Under Paste and Drop Text you can specify how Cindex deals with style and font information when pasting or dropping text that originates externally (e.g., from a PDF document). Choose whether Cindex accepts styles and fonts, styles only, or just plain text. The default is Styles Only. You can override the preference setting when actually pasting or dropping text.

Other

  • Set Bad Locator to establish what Cindex will do when a locator field is empty, contains a malformed locator (e.g., a range of pages such as 89-67), or one that falls outside an allowable range. For Warn, Cindex will accept the record when you make a second attempt to save it; for Forbid Cindex will not save the record until you correct the error.
    Set Bad Cross-Ref to determine how Cindex will behave when you enter in the locator field a cross-reference that has no valid target.
  • Cindex can check that the contents of each field of a new record match a template you have specified. Set Mismatch to Template to establish what will do if the contents of a field do not match its template. For Warn, Cindex will accept the record when you make a second attempt to save it. For Forbid, Cindex will not save the record until the field conforms to the template.
  • If you close the window from the File menu, or by clicking the close box, Cindex behaves according to your setting of On Closing Window (always saves changes, asks you about saving changes, or always discards changes). These settings do not apply when you complete editing with a keystroke: hitting CONTROL RETURN (or ENTER on the keypad) always saves the edits; hitting ESC always discards them.
