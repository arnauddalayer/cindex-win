Options for saving delimited records

To set options for saving tab-delimited records, click Options in the panel Cindex presents when you choose Save As… from the File menu.

Settings you make here control how Cindex saves tab-delimited records in a file.

  • Items under Scope define the collection of records to be saved. For more information on this see Specifying the scope of a command.
  • Cindex normally saves records in the order which they appear in the current view. Check Write in Record Number Order to make Cindex save records in record number order.
  • Cindex normally uses UTF-8 (Unicode) to encode the text of records. Click ANSI to make Cindex encode text using the Windows ANSI character set. The ANSI character set cannot represent all characters that might appear in records.
  • Cindex normally saves each record with exactly the number of fields it contains. To make Cindex save records with a minimum number of fields, enter that number at Minimum Number of Fields. If a record contains fewer then this minimum number, Cindex writes empty fields between the last subheading and the locator field.
