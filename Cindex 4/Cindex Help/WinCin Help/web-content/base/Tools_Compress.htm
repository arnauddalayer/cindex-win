Removing unwanted records

Use the Compress panel (Tools menu) to remove unwanted records from the index.

To remove records that contain no text check Remove Empty Records.

To remove deleted records check Remove Deleted Records.

To remove exact duplicates check Remove Duplicated Records.

To remove records that contain automatically-generated cross-references check Remove Generated Records.

As you build an index you might create several records that are identical except for their page references or cross-references. To simplify the organization of records, Cindex can merge the identical entries, putting all their page references and cross-references into as few records as possible, and deleting any unnecessary records that result. To consolidate references, and to remove records that are no longer needed, check Consolidate References. By default, Cindex does not consolidate references from otherwise identical records that carry different labels. To make Cindex ignore labels, check Ignore Different Labels. When this is checked, a merged record carries the label (if any) of the first of the merged set.

You need not consolidate references—Cindex always merges duplicate entries when it displays or prints the formatted index—but you can reduce the size of your index by doing so.

Cindex cannot consolidate references or remove duplicated records if the index is sorted in page number order, in which case these items are dimmed.

When Cindex removes records from an index it renumbers those that remain. You should not rely on a record having the same number before and after you compress an index. Cindex uses a record’s number to keep track of its membership in any group. Because record numbers often change when you compress an index, Cindex invalidates all its groups.
Before you consolidate references, make sure that Cindex has been set correctly to recognize the character that separates successive references when more than one appears in the locator field.
