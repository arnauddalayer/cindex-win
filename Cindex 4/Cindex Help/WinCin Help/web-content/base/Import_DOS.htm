Importing indexes from Cindex for DOS

Cindex for Windows can open and use these files made by the legacy program Cindex for DOS:

  • Index files (those with the .ndx filename extension) made by Cindex Version 6.0 and higher.
  • Files containing backup records (those with the .dat filename extension) made by any version of Cindex for DOS.

Cindex for Windows cannot open directly an index created by Cindex for DOS version 5 or below. To use such an index, you must open it with the version of Cindex used to create it, then use the command WRITE to export the index records. Use Cindex for Windows to create a new index, then import the records you have written.

To open and convert an index made by Cindex for DOS, start Cindex, then choose Open from the File menu. From the Files of type drop-down list choose Indexes to display DOS indexes, then choose the file you want to convert.

When Cindex for Windows converts an index made by Cindex for DOS, with the exceptions noted below it preserves format and layout settings, translates all text attributes, and converts special DOS characters into their Windows counterparts. When importing index records from a DOS backup file (one with the .dat filename extension), Cindex translates attributes and special characters into their Windows counterparts.

Important Conversion Issues

If Cindex for Windows encounters a character it cannot translate when converting a DOS index, or reading records from a DOS backup file, it will represent that character by the Unicode unknown character � and it will ‘mark’ the record that contains the character. If Cindex has marked any records it will alert you when it has finished the conversion. You can use the Find command (Edit menu) to find any marked records

The \g…\G and \h…\H attribute codes, available for use in Cindex for DOS, have no counterparts in Cindex for Windows. When Cindex for Windows opens a .ndx or .dat file it checks for the presence of these attributes. If they have been used in the file, Cindex will warn you and invite you to translate them into distinctive fonts. If you decline to translate these attribute codes, they will be discarded from the records during conversion.

Cindex for Windows does not convert record groups that belong to the index being converted. You will need to re-create these after conversion.
