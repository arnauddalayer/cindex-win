Checking page references

Checking page references is available as part of a comprehensive check via Check Index… (Tools menu).

Open the panel, click the References tab, then make settings under Page References.

Check Missing to find records that lack a cross-reference or a page reference. When checking for a cross-reference, Cindex will accept one in a subheading (as an alternative to the locator field) if you have unchecked Recognize cross-references in locator field only in Reference Syntax…(Document menu).

At More than n undifferentiated enter the maximum number of locators acceptable in an entry.

Check Overlapping ranges to find cases where a single reference is duplicated, or a single reference or a range of references overlaps another range. When dealing with references that contain non-numerical suffixes, (e.g., 37t), or are styled (e.g., 522), Cindex examines them for potential overlap only if the suffix is the same, or the style is the same. When dealing with compound (multi-part) locators (e.g., 25a:64), Cindex examines them for potential overlap only if all segments but the last are identical.

When producing a formatted Index, Cindex can automatically suppress the redundant parts of overlapping references (see settings under Page References in the Document menu), so you need not check for them unless you think they need attention as potential errors.

Check On headings above lowest if you want to identify entries in which any heading level above the lowest subheading has a reference attached to it.

If Cindex finds errors it displays them in the check results. Each entry in the list identifies the source record and the nature of the error.

Click any item in the list to bring its record into view in the main document window; double-click the item (or hit RETURN) to open a window in which to edit its record.
