Working with XML records

For interchange with other indexing software, Cindex can export and import index records as tagged elements in XML documents. These documents identify key elements of records, such as required last text fields, and locator fields, and include complete information about fonts and text styles used in records.

To save an index as XML records, choose Save As… from the File menu then from the File Format drop-down list choose XML Records. Cindex automatically provides the appropriate filename extension (.ixml) when it saves the document.

Before saving XML Records, Cindex checks the index for records that contain mismatched < > or mismatched { } within a field. If any records contain mismatched < > or { }, Cindex will warn you and will display the records in a group. You will need to correct the mismatches before you can export the records.

Cindex provides two ways to use records in XML format: you can create a new index directly from the XML Records file, or you can import the records into an existing index.

To add the contents of the XML Records file to an existing index, choose Import… from the File menu, then select the document you want.

To create a new index from XML Records, choose Open… from the File menu, then select the document by name from the displayed list. You can also do this through the Windows Explorer by double-clicking the document’s icon. If no index with the name of the XML document exists in the folder that document Cindex will automatically create a new one with the name of the document and will import the records. If an index with the name of the document already exists, Cindex will present a panel through which you should choose the name and location of the index to be created.
