Checking the index

Cindex can undertake a comprehensive check to identify many kinds of errors or potential errors in index entries.

Use the Check Index… panel (Tools menu) to specify which checks Cindex should perform on the index. The three tabs in the panel group checks of different kinds:

  • Basic: examines individual fields of records, and detects questionable punctuation and badly-formed words.
  • Headings: compares headings across records, and checks for inconsistencies, and such things as orphaned subheadings.
  • References: examines page references and cross references, and detects incorrect or badly-formed references of the specified type.
Check Index is available only when the index is sorted alphabetically.

When you click OK, Cindex undertakes all the checks you have enabled (it also remembers your settings from one invocation to the next).

If Cindex finds errors it displays them in a list. Each entry in the list identifies the source record, and the nature of the error.

Click any item in the list to bring its record into view in the main document window; double-click the item (or hit RETURN) to open a window in which to edit its record.

When a particular error (e.g., a space missing before an opening parenthesis) occurs identically in a succession of records, Cindex reports only the first occurrence.

When an error is related to page references, and more than one record contributes references to the set of references, the record number identified in the report will be that of the first record in the set.
