Verifying cross-references

Verifying cross-references is available as part of a comprehensive check via Check Index… (Tools menu).

Open the panel, click the References tab, then check Verify Cross-References.

Cindex normally accepts a shortened form of the main entry as a legitimate cross-reference. To insist that Cindex find an exact match (including accent and letter case) between a cross-reference and its target entry check Require Exact Matches. When you insist on exact matches, Cindex requires only that the text of the cross-reference matches that of the target; styles and fonts are ignored.

Cindex can flag cross-references made to fewer than a certain number of entries. (Cindex must always find at least one target of a cross-reference, otherwise it flags a reference as missing.) Set Require at least n matches to the minimum number of target entries Cindex must find for each cross-reference.

If Cindex finds errors it displays them in the check results. Each entry in the list identifies the source record, the intended target, and the nature of the error, which can be one or more of:

missing target The cross-reference points to a heading that is not in the index, or to a heading that lacks a locator.
inexact match The cross-reference and its target differ in letter case or accent.
circular or open The cross-reference points to another cross-reference.
too few targets The cross-reference points to a heading with fewer than the stipulated minimum number of subentries.
wrong type A See cross-reference should have been a See also cross-reference (or vice-versa). This indicator can accompany any of the others.

Click any item in the list to bring its record into view in the main document window; double-click the item (or hit RETURN) to open a window in which to edit its record.
