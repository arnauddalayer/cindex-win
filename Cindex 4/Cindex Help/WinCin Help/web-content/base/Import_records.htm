Importing records

To import records from the following kinds of documents:

  • Any version of Cindex for Windows (filename extension .cdx)
  • Cindex for Mac version 3 and higher (filename extension .ucdx)
  • Cindex for DOS version 6 or higher (filename extension .ndx).

choose Open… from the File menu, or double-click the document's icon. From the opened document, drag (or copy) the records you want into the destination index.

To import records from an index created by Cindex for Mac version 2 or lower, the index should be in archive format. Choose Import from the File menu to import records into the destination index.

You should not use the archive format when moving a Cindex 4 index between Windows and Mac. The archive format does not allow representation of all Unicode characters.

Cindex can import records in several variants of a standard delimited format.

  • Plain Text — records contain plain text (not style or font information). Fields are separated by tab characters.
  • Sky Text — records are in the export format produced by Sky Index.
  • Macrex MBK — records are in the backup format used by the indexing program Macrex.
  • DOS Cindex data — records are in the legacy format used by Cindex for DOS and contain information about styles but not fonts. Fields may be separated by tab characters, or enclosed in quotation marks.

Choose from the Files of Type drop-down list the format of the records you want to import. Cindex can generally recognize automatically how records are formatted, and you should usually choose All Readable. If Cindex fails to identify the format of records, it will warn you.
