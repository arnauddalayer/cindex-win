Setting record structure

To examine or modify the structure of index records (their sizes, the number and names of fields, and any restrictions on what fields may contain) choose Record Structure... from the Document menu.

  • To specify the maximum number of characters a record may contain (< 2000), enter the number at Maximum Characters in Record. If the index already contains records, the smallest number you can set is shown in the box immediately to the right.
  • To specify the minimum number of fields a record may contain (>= 2), choose from the Min drop-down list.
  • To specify the maximum number of fields a record may contain (<= 16), choose from the Max drop-down list.
  • To make every record have a special text field that immediately precedes the locator (regardless of how many normal subheadings the record contains), check Subhead Preceding Locator is Required. This field might be used for notes, or for storing URLs. If you wish, this field can be suppressed in formatted output, and ignored in sorting entries. If displayed, the special last subheading is formatted as the lowest level subheading permitted in the index. To enable this special field you must have set the minimum number of fields to be at least 3. When you have designated the last subheading as a required field, Cindex indicates this with an asterisk (*) in the tab for that field.
  • To specify the name and properties of an individual field of a record, click the tab corresponding to the field (First corresponds to the Main heading, etc.; the locator field is always last). Enter at Name the name Cindex will use to identify the field. Cindex displays this name to identify a field to you in all any operation the deals with fields.
  • Enter at Minimum Chars and Maximum Chars the minimum and maximum number of characters that the field may contain. To impose no minimum or maximum, set the corresponding number to 0. If the index contains records, Cindex displays (in the box on the right) that largest number of characters in that field in any record.
  • Enter at Match a template that describes the permissible content of the field. The template should be a pattern that the contents of the field must match before Cindex will accept the record in the index. Leave Match blank to impose no restriction on the contents of the field. If you specify a template, how Cindex uses it depends on settings you make by choosing Preferences... from the Edit menu.

If an index already contains records there are restrictions on what you can change:

  • You cannot set the maximum number of characters in a record to be fewer than are contained in the longest record.
  • You cannot set the maximum number of fields to be fewer than are contained in the record with most fields.
  • You cannot set the minimum length of a field.
  • You cannot set the maximum length of a field to be less than the largest number of characters in that field in any record.

If you make settings when no index is open for work, these become the default ones that Cindex uses for new indexes when you create them.
