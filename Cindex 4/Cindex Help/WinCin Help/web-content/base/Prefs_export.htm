Exporting formatted documents

You can specify the default method Cindex will use to form subheading indents in documents that contain formatted indexes. To inspect or change settings, choose Preferences... from the Edit menu, then click the Formatted Export tab.

  • Choose Style to include style definitions that programs such as Microsoft Word and QuarkXPress and InDesign can use to reproduce the layout you have worked with in Cindex.
  • Choose Tab if you want Cindex to insert one tab stop for each level of subheading.
  • Choose Other (and provide a single character) to make Cindex insert one such character for each level of subheading.

Set Include sorting information when embedding entries to determine whether Cindex will provide special sorting information when it embeds an entry in a Rich Text Format (RTF) document.

When you save a formatted index as Plain Text, Cindex can use Unicode (UTF-8) or Windows ANSI character encoding. UTF-8 is recommended, and is the default. Click your choice at Encode Plain Text Documents.

When you save the formatted document you can override these default settings by clicking Options… in the Save As… dialog box.
