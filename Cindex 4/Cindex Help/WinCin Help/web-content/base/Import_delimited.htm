Importing records from delimited text

Cindex can import records that have been prepared elsewhere (for example, by a word-processing program or text editor, or database program).

To import records into an existing index, choose Import from the File menu. Cindex assumes that documents with the filename extension ‘.txt’ are potential sources of delimited records, and the Open panel shows all such files as readable.

Records must conform to a simple specification:

  • If the document begins with the standard signature of a UTF-8 document, Cindex will recognize the Unicode character encoding. Otherwise Cindex assumes that characters are from the Windows ANSI character set.
  • Each record must be contained in a paragraph terminated by a carriage-return character, or a line-feed character, or a carriage-return/line-feed character pair.
  • Each record can contain up to fifteen tab-separated fields that hold the text of headings, plus a field that contains the locator.
  • The fields of the record must be separated from each other by a ‘tab’ character, and the locator field, which contains page references or cross-references, must be the last field.
  • Cindex assumes that the last field in any record is the locator field, so you should ensure that any record that has no locator (page reference or cross-reference) has a blank last field (i.e., one with nothing after the last tab character). You need not include empty fields between headings and the locator field: if you import records into an index in which records have a fixed number of fields, Cindex will automatically add empty fields when necessary.

If Cindex encounters any of the characters { } < > ~ \ in the imported text, it places a \ before it (e.g., { in the imported text becomes \{ in the Cindex record) to ensure that it is treated as a literal character, rather than a special function character.
