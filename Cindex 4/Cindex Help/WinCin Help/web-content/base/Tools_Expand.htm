Expanding index entries

Use Expand (Tools menu) to unpack references from records that contain more than one, and generate new records as necessary, so that none in the index contains more than one page reference or cross-reference. You should have no more than one reference in each record if you intend to sort index entries by order of page number.

You can use Expand only when the main index view shows All Records.

When Cindex expands references, it normally recognizes successive page references separated with a comma, and successive cross-references separated with a semicolon. If you have used different characters to separate these two kinds of references, you should ensure that Cindex knows what they are.

When Cindex expands an index it invalidates any record groups that belong to the index.
