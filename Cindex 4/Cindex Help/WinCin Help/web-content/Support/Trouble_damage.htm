Recovering from damage

If some computer malfunction causes damage to an index, Cindex will report that when it tries to read records from the index.

If Cindex reports an error, you should close the index, then try to repoen it. Each time you open an index, Cindex checks its integrity, and if it finds errors will recommend that you let it repair the index.

Cindex can repair many but not all errors. If the text of a record is damaged and contains illegal characters, Cindex will replace each illegal character with @.

When Cindex repairs a record it marks it, so that you can later review it.
